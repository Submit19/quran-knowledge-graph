## Recommended set — gap analysis

### 1. Composite range index on (Verse.surah, Verse.verseNum)

**Status: MISSING**

The task spec calls for a `verse_surah_ayah` composite RANGE index. Currently:
- `verse_surah` RANGE on `surah` (single property) — exists
- `verse_id` RANGE on `verseId` — exists
- `verse_ref` RANGE on `reference` — exists

Queries like `MATCH (v:Verse {surah: $s, verseNum: $vn})` or range scans across a surah
cannot use a composite index and fall back to `verse_surah` + row-level filter on `verseNum`.
At 6,231 verses this is ~2ms today (confirmed by slow-query-log tick), but a composite
index eliminates the filter step entirely and is free to add.

**Verdict:** Add. Low urgency (small dataset) but cheap and correct.

### 2. Relationship-property indexes on MENTIONS.from_tfidf / to_tfidf

**Status: MISSING**

All 41,062 MENTIONS edges carry both `from_tfidf` and `to_tfidf` properties (confirmed).
No index on either. Range filter queries (`WHERE m.from_tfidf > 0.5`) scan all 41K edges.

Not currently a hot path (most retrieval goes vector/BM25), but required before any
TF-IDF-weighted traversal mode.

**Verdict:** Add both. Moderate priority.

### 3. MENTIONS.score relationship-property index

**Status: MISSING**

`MENTIONS.score` is used by `search_keyword` Cypher: `WHERE m.score > 0.1`. No index.
41K edges scanned per call. Current p50 ~2ms (acceptable) but index would make it ~0ms.

**Verdict:** Add. Low cost, moderate benefit.

### 4. All vector indexes

**Status: PRESENT and correct**

- `verse_embedding` (384d MiniLM — legacy)
- `verse_embedding_m3` (1024d BGE-M3 EN — active)
- `verse_embedding_m3_ar` (1024d BGE-M3 AR — active)
- `query_embedding` (384d MiniLM — **dimension mismatch with verse_embedding_m3**)

The `query_embedding` mismatch is a known gap tracked by task `refresh_query_embedding_to_bge_m3` (p85).
All are ONLINE, COSINE, HNSW (m=16, ef_construction=100), quantization enabled.

### 5. Full-text indexes

## Migration script

```cypher
-- ================================================================
-- QKG Index Migration — 2026-05-11
-- Run against quran database. All use IF NOT EXISTS — safe to re-run.
-- ================================================================

-- 1. Composite range index on Verse(surah, verseNum)
CREATE INDEX verse_surah_ayah IF NOT EXISTS
  FOR (v:Verse) ON (v.surah, v.verseNum);

-- 2. MENTIONS relationship-property index on from_tfidf
CREATE INDEX mentions_from_tfidf IF NOT EXISTS
  FOR ()-[m:MENTIONS]-() ON (m.from_tfidf);

-- 3. MENTIONS relationship-property index on to_tfidf

  FOR ()-[m:MENTIONS]-() ON (m.to_tfidf);

-- 4. MENTIONS relationship-property index on score
```
