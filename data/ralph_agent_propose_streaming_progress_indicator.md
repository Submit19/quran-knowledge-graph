* The progress bar sits just below the streaming markdown area.  
* It shows **current index / total tools**, the **tool name**, and a **short summary** (the `summary` field from the `tool` SSE).  
* When a tool finishes, the bar advances.  
* If no tools are fired (cache hit), the bar stays hidden.  
* Errors are displayed inline, replacing the bar until the next tool event or the final `done` event.

### 2.3. JS – extend the existing SSE handler (in `index.html` or its bundled script)

```js
// State for progress indicator
let toolCount = 0;
let totalTools = 0;
let toolNames = [];   // optional, for future UI

function showToolProgress() {
  const el = document.getElementById('tool-progress');
  el.classList.remove('hidden');
}

function hideToolProgress() {
  const el = document.getElementById('tool-progress');
  el.classList.add('hidden');
}

// Called when a 'tool' event arrives
function handleToolEvent(data) {
  // data = {name, args, summary}
  if (totalTools === 0) {
    // first tool event – we don't know total yet, assume unknown.
    // We'll update total when we receive the next 'tool' or 'done'.
    totalTools = 1; // provisional, will be corrected later
  } else {
    totalTools = Math.max(totalTools, toolCount + 1);
  }
  toolCount += 1;

  // Update UI
  document.getElementById('tool-count').textContent = `${toolCount} / ${totalTools}`;
  document.getElementById('tool-name').textContent = `${data.name}: ${data.summary}`;
  const percent = Math.round((toolCount / totalTools) * 100);
  document.getElementById('tool-bar').style.width = `${percent}%`;

  showToolProgress();
}

// When the server sends a final 'done' or an 'error', hide the bar
function finalizeProgress() {
  hideToolProgress();
  toolCount = 0;
  totalTools = 0;
}

// Hook into the existing EventSource loop
const evtSource = new EventSource('/chat?...');
evtSource.addEventListener('tool', e => {
  const payload = JSON.parse(e.data);
  handleToolEvent(payload);
});
evtSource.addEventListener('error', e => {
  // optional: show error message in place of progress bar
  const err = JSON.parse(e.data);
  document.getElementById('tool-status').textContent = `Error: ${err.d}`;
  // keep bar visible until next tool or done
});
evtSource.addEventListener('done', e => {
  finalizeProgress();
});
```

*If the client bundles JS elsewhere, copy the three functions (`showToolProgress`, `hideToolProgress`, `handleToolEvent`, `finalizeProgress`) into that module and register the listeners.*

## 3. State Machine  

| Current State | Incoming SSE | Action | Next State |
|---------------|--------------|--------|------------|
| **Idle** (hidden) | `tool` | `totalTools = 1` (or keep existing), `toolCount = 1`, update UI, show bar | **Running** |
| **Running** | `tool` | `toolCount += 1`; `totalTools = max(totalTools, toolCount)`; update UI | **Running** |
| **Running** | `error` | Show error text inline (optional), keep bar | **Running** |
| **Running** | `done` | Hide bar, reset counters | **Idle** |
| **Running** | `graph_update` / `text` / `verification` | No UI change | **Running** |
| **Idle** | `done` (cache hit) | Ensure bar stays hidden | **Idle** |

*When the server knows the total number of tools ahead of time (e.g., it sends a `tool_count` field in the first `tool` event), the client can set `totalTools` immediately. The fallback logic above works even when the total is unknown until the stream ends.*

## 4. Edge‑Case Handling  

| Situation | Handling |
|-----------|----------|
| **Cache hit (no tool events)** | The progress bar never becomes visible; UI remains unchanged. |
| **Error occurs mid‑stream** | Show the error message in the status line; keep the bar at its last percentage so the user sees where the failure happened. On the next `tool` event (if any) the bar resumes normal updates. |
| **≥10 tools** | The bar scales automatically; the numeric count (`10 / 12`) remains readable. If the count exceeds two digits, the layout still fits because the status line uses `flex`‑like inline layout. |
| **Tool count unknown** (server does not send total) | Use the highest observed `toolCount` as provisional total; the bar will reach 100 % on the final `done` event (set `totalTools = toolCount` in `finalizeProgress`). |
| **Rapid succession of tool events** | UI updates are cheap (DOM text + width change); the `transition` ensures smooth visual progress. |
| **User aborts chat** (client closes SSE) | `hideToolProgress()` is called in the `onclose` handler (add `evtSource.onclose = finalizeProgress;`). |
