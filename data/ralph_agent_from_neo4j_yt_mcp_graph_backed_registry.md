# Graph‑Backed Tool Registry for MCP (Neo4j + YT)

## Overview
This document describes the implementation of a **graph‑backed tool registry** that ranks tools for the **MCP (Multi‑Component Processor)** startup list using historical usage signals stored in Neo4j. The registry draws on the `RETRIEVED` edge counts already persisted by `reasoning_memory.py`. No new infrastructure is required; we simply query the existing graph, compute usage scores, and expose a deterministic ordering for the MCP server.

## Data Source

| Node Type | Relationship | Property Used |
|-----------|--------------|---------------|
| `Tool`    | `RETRIEVED`  | `count` (integer) – number of times the tool was retrieved during reasoning |
| `ToolGroup` (optional) | `BELONGS_TO` | – |

The `RETRIEVED` edges are created each time a tool is selected by the reasoning engine. The `count` property is incremented atomically, giving us a reliable usage metric.

## Core Algorithm

1. **Connect to Neo4j**  
   ```python
   from neo4j import GraphDatabase

   driver = GraphDatabase.driver(
       uri="bolt://localhost:7687",
       auth=("neo4j", "YOUR_PASSWORD")
   )
   ```

2. **Query Usage Counts**  
   ```cypher
   MATCH (t:Tool)-[r:RETRIEVED]->()
   RETURN t.name AS tool_name, r.count AS usage_count
   ORDER BY usage_count DESC
   ```
   - Returns every registered tool together with its cumulative retrieval count.
   - Ordering is performed in the query to minimise post‑processing.

3. **Normalize (optional)**  
   If a relative ranking is required (e.g., for UI weighting), compute a normalized score:
   ```python
   max_count = max(row['usage_count'] for row in results)
   for row in results:
       row['score'] = row['usage_count'] / max_count
   ```

4. **Expose Registry API**  
   ```python
   from fastapi import APIRouter

   router = APIRouter()

   @router.get("/mcp/tools")
   async def list_tools(limit: int = 20):
       with driver.session() as session:
           records = session.run(
               """
               MATCH (t:Tool)-[r:RETRIEVED]->()
               RETURN t.name AS name, r.count AS usage
               ORDER BY usage DESC
               LIMIT $limit
               """,
               limit=limit
           )
           return [{"name": r["name"], "usage": r["usage"]} for r in records]
   ```
   - The endpoint returns the top‑`limit` tools sorted by historical usage.
   - The same endpoint is used by the MCP server during its **startup** phase to build the initial tool list.

## Integration with `from_neo4j_yt_mcp_balanced_tool_grouping`

The balanced grouping task (p78) expects a **pre‑sorted list** of tools. By providing the ordered list from the registry, the grouping algorithm can:

2. Ensure that low‑usage tools are still represented (by cycling through the sorted list).

## Edge Cases & Resilience

| Situation | Handling |
|-----------|----------|
| **No `RETRIEVED` edges** (new tool) | Return `usage = 0`; tool appears after all used tools. |
| **Stale counts** (e.g., after a data purge) | Periodic background job (`recalc_usage_counts.py`) can recompute counts from raw reasoning logs. |
| **Concurrent updates** | Neo4j’s native ACID guarantees ensure `r.count` increments are atomic. |
| **Large tool catalog** | Pagination (`skip`/`limit`) is supported; the API can stream results if needed. |

## Maintenance Script (Optional)

A lightweight script to **re‑seed** usage counts from the reasoning log files:

```python
#!/usr/bin/env python3
from neo4j import GraphDatabase
import json, glob

driver = GraphDatabase.driver("bolt://localhost:7687", auth=("neo4j", "YOUR_PASSWORD"))

def rebuild_counts():
    usage = {}
    for file in glob.glob("logs/reasoning/*.json"):
        with open(file) as f:
            data = json.load(f)
            for entry in data.get("retrieved_tools", []):
                usage[entry] = usage.get(entry, 0) + 1

    with driver.session() as s:
        for tool, cnt in usage.items():
            s.run(
                """
                MERGE (t:Tool {name: $name})
                MERGE (t)-[r:RETRIEVED]->()
                SET r.count = $cnt
                """,
                name=tool, cnt=cnt
            )

if __name__ == "__main__":
    rebuild_counts()
```

Running this script is **not required** for normal operation; it is provided for recovery scenarios.
