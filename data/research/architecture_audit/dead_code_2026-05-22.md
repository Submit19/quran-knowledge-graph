## 8 · Test-suite hygiene

### 8a · `tests/` directory

- **No stale `@pytest.mark.xfail` markers.** Grep `tests/` confirms zero hits — the xfail-then-flip TDD discipline (per `CLAUDE.md` Engineering discipline) is being followed correctly. The single docstring mention in `tests/test_answer_cache_rerank.py` references the pattern but no live marker remains; that test passed after `5cc0e6c` shipped the implementation.
- **One `@pytest.mark.skipif`:** `tests/test_cache_coverage_shuaib_alias.py:62` — guards on `data/answer_cache.json` being present (gitignored fixture). Correct behaviour for a fresh worktree.
- **No all-skipped or all-xfail files.**

### 8b · Root-level `test_*.py` (collection contamination — addressed)

- `test_va_impact.py` (lines ≈ 240) and `test_verse_analysis.py` (lines ≈ 260) at the repo root are manual A/B harnesses, NOT pytest tests. They `import requests` at module load and read `.env` eagerly.
- **Status:** `pyproject.toml` pins `testpaths = ["tests"]` (commit `625b404`), so `pytest` invoked bare won't collect them. CLAUDE.md §"Fresh-worktree setup" warns about this.
- **Residual risk:** the filenames still match the `test_*.py` glob, so any external tool (CI shard, IDE collector, `unittest discover`) that does its own collection will trip. Renaming to `harness_va_impact.py` / `harness_verse_analysis.py` removes the trap without changing semantics. **Flag for Pass 5 quick wins.**

### 8c · `tests/regression/` is healthy

7 files, all real regression tests pinning bug fixes from Phase 3b (whitespace cluster, run_cypher denylist, SSE worker leak, tool-cache TTL race, typed-edges concat, mid-turn stop-poll, compress_tool_result full_coverage kwarg). All passing per the 209+1 baseline. No dead content.
