# Gemini Backend — End-to-End Smoke Test

**Date:** 2026-06-08
**Server:** `app_free.py --gemini --port 8090` (SEMANTIC_SEARCH_INDEX=verse_embedding_m3,
RERANKER_MODEL=BAAI/bge-reranker-v2-m3), Neo4j up (6,234 verses).
**Backend confirmed live:** `/model-status` → `{"state":"ready","model":"gemini-2.5-flash","backend":"gemini"}`;
`/model-info` → `{"model":"gemini-2.5-flash","backend":"gemini","cost":"free"}`;
badge HTML renders `⏳ connecting to gemini (gemini-2.5-flash)…`.

Each query went through the real `/chat` SSE agentic loop. Tool calls were
dispatched via `chat.dispatch_tool`; citations are `[surah:verse]` refs parsed
from the streamed answer.

## Headline result

**Gemini drives the agentic loop correctly, including parallel multi-tool
function-calling.** Query 1 ran entirely on Gemini and produced a rich, grounded
answer. Queries 2–4 hit the Gemini **free-tier quota (20 requests/window,
`RESOURCE_EXHAUSTED`, "retry in ~33–43 s")** and the **fallback chain degraded
gracefully to local `qwen3:14b`**, which completed each query with correct tool
use and citations. Both the Gemini path and its fallback are therefore validated
in one run.

> The free-tier limit is low (20 req/window) and a single thematic query makes
> ~6–9 LLM calls (one per turn; Gemini batches multiple tool calls per turn).
> Query 1 alone consumed most of the window, so 2–4 fell back. This is exactly
> the rate-limit case the task brief anticipated.

## Tool-translation edge cases hit
1. **Dashed Arabic root** (`ش-ك-ر`) — model artefact, not a translation bug; the
   tool rejects it and the model self-corrects. No change needed.
2. **Parallel function-calling** — Gemini returns multiple `functionCall` parts
   in one response; `_from_gemini_response` already emits one `tool_calls` entry
   per part, and the loop dispatches them all. Worked first try.
3. **`functionResponse.response` must be an object** — tool results are JSON
   strings; `_to_gemini_contents` wraps non-dict payloads as `{"result": …}` and
   passes dict payloads through. Confirmed by the multi-turn live round-trip.

## Known limitation / follow-ups
- **Free-tier quota is the binding constraint** for sustained agentic use; expect
  frequent fallback to local Ollama. For uninterrupted Gemini eval (e.g. the
  composer-rewire obedience comparison) use a paid key or pace requests to
  ≤20/window. A deliberate spaced retry of Q4 after a 75 s pause **still** 429'd
  on turn 1 — i.e. query 1's burst exhausted the window for the rest of the
  session, so the reset is longer than the per-error "retry in ~33 s" hint
  suggests (the 20-request limit appears to have a longer/daily component).
  Operators should run the manual Gemini smoke with a paid key or a fresh daily
  quota.
- **429 retries count against the quota.** `_gemini_chat` retries 429 up to 3×;
  since the retry-in window (~33 s) far exceeds the backoff (1+2 s), the retries
  never succeed and each consumes a request. A cheap improvement: on a 429 whose
  `retryDelay` exceeds the backoff budget, fall back immediately instead of
  retrying. Kept as-is for now to match the brief's "retry-with-backoff then
  propagate" spec.
- **Startup banner** still prints "local Ollama" even under `--gemini`
  (`startup_banner.format_startup_banner` has no Gemini parameter). Cosmetic —
  the line below it correctly prints `Default backend: gemini :: gemini-2.5-flash`.
